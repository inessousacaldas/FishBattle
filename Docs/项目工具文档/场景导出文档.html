寻路导出

  • 打开Unity上方item 中的 GameObject/CreateEmpty
  • 在新建的GameObject中添加脚本 PathFinder
  • 在新添加的脚本中选，AddNewGraph -> NavMeshGraph
  • 将制作好的网格FBX拉到上图中，SourceMesh中。

  • 点击Scan，扫描后可以在navMeshGraph中调整位置缩放。
  • 调整完成后，选择Svae&Load -> Save to file
![Alt text](./1508410365470.png)
  • 保存到Assets\GameResources\ArtResources\SceneConfig\Navmesh这个文件夹,命名规则为navmesh_scene_场景编号.bytes

客户端场景导出

  • WorldStage根节点下的所有是Prefab的GameObject都会自动被导出并设置BundleName

    ExportTool/ExportScene进行导出,导出的文件目录为Assets\GameResources\ArtResources\SceneConfig\LightMap
    Assets\GameResources\ArtResources\SceneConfig\SceneGo
    分别为光照贴图数据文件和场景信息数据文件

  • 注意检查WorldStage下是否有 “场景编号_way” 和 “场景编号_way_all” 等2个meshCollider

  • 场景导出后，导出的prefab上都会挂有SceneGoInfoComponent组件，可用于调整包围盒的大小
  • 注意日志中提示未能计算包围盒大小的物体，需要在场景中手动调整该物体包围盒

场景漫游:

  • 打开需要漫游的Scene
  • 将Preview下的Capsule拉到场景中
  • 运行Unity
  • 注意：场景漫游需要依赖场景导出生成的SceneGoInfoComponent中的数据，但不需要依赖导出的文件

如何生成Capsule Prefab

  • 在场景内创建或拖入一个对象.挂上TravelScene脚本
  • 在该对象下再挂一个带相机的子节点
  • 运行场景，用键盘aswd 控制移动

战斗场景雾效处理

服务器寻路数据导出

  • 运行游戏，进入需要导出的场景
  • 进入场景后，用ExportTool/ExportServerNavMesh 导出，导出目录在Docs/NavigationArea下。
  • 如果没有导出数据，注意查看 “场景编号_way”和 “场景编号_way_all” 2个MeshCollider的Layer层级是否为Terrain

FAQ

  • 导出场景后物体在Game窗口中不显示，在Scene窗口中显示

    • 查看该物体的Layer，是否为Default
    • 如果不是则看该物体的Prefab Layer是否正确。
    • 如果正确则查看Prefab中是否有Collider组件，有则删除（含有Collider组件的物体会被代码设置Lyaer）。

  • 直接打开Scene的表现和运行时加载的场景表现不一致

    • 查看该物体在Scene中是否有修改没有保存到Prefab中
    • 具体方法是看该物体的面板，是否有出现粗体字的字符。（在Unity中Prefab拉进来的物体出现粗体字表示该属性与Prefab中的不一致）
